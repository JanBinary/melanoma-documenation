2.1 Grundlagen Computer Vision

Wie kann ein Computer sehen?

Faltung (Convolution)

\[ I^{\ast} (x, y) = \sum^n_{i=1} \sum^n_{j=1}{I(x-i+a, y-j+a)K(i, j)} \]

Aufmerksamkeitsmechanismus (Attention)

\[ \text{Attention}(Q, K, V) = \text{softmax}\left( \frac{QK^\top}{\sqrt{d_k}} \right) V \]

  • a offset zur Kernelmitte (a = (n + 1)/2) -> Zur Verschiebung des Filters um den Pixel herum
  • Umfrage (wer kennt sich mit ViT aus)
  • Q (Query) : Was will ich wissen?
  • K (Key) : Was ist dafür relevant?
  • V (Value) : Welchen Kontext trage ich bei?
  • T : Transposition (QKT ergibt eine quadratische Matrix)
  • dk : Dimensionen der Key Vektoren

Faltung (in CNNs)

  • Ein Filter (Kernel) durchläuft das Bild stückweise um Muster zu erkennen
  • Frühere Schichten erkennen einfache Merkmale wie Kanten, Ecken, Farbübergänge
  • Spätere Schichten kombinieren einfache Muster zu komplexeren Formen
  • CNNs erkennen Merkmale unabhängig von ihrer Position im Bild

Funktionsweise Faltung [3]

Abbildung 3: CNN Architektur (aus Medium)

Achtsamkeitsmechanismus (in ViTs)

  • Bild wird in kleine Patches zerlegt, die wie Wörter verarbeitet werden
  • Jedes Patch achtet auf andere Patches und gewichtet deren Einfluss (Self-Attention)
  • Modell lernt Beziehungen zwischen Bildteilen unabhängig von deren Abstand
  • ViTs erkennen globale Zusammenhänge frühzeitig, nicht nur lokale Muster

Funktionsweise Achtsamkeit [4]

2.2 Was passiert bei der Faltung

  • 🔍 Ein Filter \(K\) überlagert das Bild \(I\) und blickt auf einen \(n \times n\) Bereich
  • ➕ Überlappende Werte werden multipliziert und aufsummiert
  • 🎯 Ein einzelner neuer Wert entsteht \(I^{\ast} (x, y)\)

\[ I^{\ast} (x, y) = \sum^n_{i=1} \sum^n_{j=1}{I(x-i+a, y-j+a)K(i, j)} \]

2.3 Anwendung der Faltung in CNNs

  • ✅ Je nach Wahl des Kernels \(K\) werden unterschiedliche Merkmale hervorgehoben
  • 🏎️ Der Kernel \(K\) bewegt sich stückweise über das komplette Bild \(I\)
  • 🏃🏿‍♂️‍➡️ Die Schrittweite des Kernels ist variabel und wird als Stride bezeichnet
  • 🅾️ Padding ist möglich (0 Werte am Rand der Matrix)

2.4 Begriffsklärung Convolution

  • Unterschied: Bei depthwise ein Kernel pro Kanal
  • Normale Convolution ist deutlich Rechenaufwänder (jeder Filter auf alle Kanäle)

  • Eingehen auf Faltungsprozesse in dem CNN Beispiel
  • Resultat der Faltung sind Feature Maps

2.5 Pooling in CNNs

  • ⬇️ Reduziert die Bildgröße durch Zusammenfassen benachtbarter Pixel
  • 💻 Verringert Rechenaufwand und Anzahl der Paramter
  • 🟰 Ziel ist Translationsinvarianz (\(f(A) = f(A + t)\))
  • 🏃🏿‍♂️‍➡️ Ein Fenster bewegt sich stückweise über das komplette Bild (ähnlich zur Faltung)
  • Translationsinvarianz: Ein Objekt wird unabhängig von seiner Position erkannt
  • Vorteil: Größe der Matrizen wird reduziert (Rechenaufwand)

Arten von Pooling

  1. Max-Pooling: Gibt den höchsten Wert im Fenster zurück
  2. Average-Pooling: Gibt den Durchschnittswert aller Werte im Fenster zurück
  3. Global Pooling: Aggregiert das gesamte Bild (Fenstergröße = Bildgröße)
  • Bei Global Pooling immer Vector als Resultat

  • Eingehen auf Pooling (kleine Feature Maps)
  • Rückschluss zu Translationsinvarianz

2.6 Aktivierungsfunktionen

  • 📏 Werden nach Faltungen / Linearkombinationen eingesetzt
  • 🎢 Ohne Aktivierung wäre ein tiefes Netz nur eine lineare Funktion
  • 📊 Helfen beim Lernen von Mustern und Entscheidungsgrenzen
  • 💯 Werden punktweise auf jedes Pixel / Feature angewendet
  • Linearkombination: Gewichtete Summe von Vectoren
  • Werden auf FeatureMap punktweise eingesetzt
  • Werden auch in NNs verwendet
  • Es gibt Unterschiedliche Funktionen je nach Anwendungsfall

Rectified linear unit (ReLU)

  • ℹ️ Einfach, schnell, sparsamer Gradient
  • 🎯 Eliminiert negative Werte

\[ f(x) = \text{max}(0, x) \]

  • Beispiel von Bildern
  • Lineare Funktion

Sigmoid

  • ℹ️ Glättet Werte auf kontinuierliche Skala
  • 🎯 Skaliert Eingaben in den Bereich (0, 1)
  • 🔻 Kann bei großen |x| zum Gradient-Verschwinden führen

\[ f(x) = \frac{1}{1 + e^{-x}} \]

  • Wird dort eingesetzt wo Werte zwischen 0 und 1 gebraucht werden
  • Nicht Robust (führt zu Problemen mit Gradienten)

Softmax

  • ℹ️ Für Klassifikation mit mehreren Klassen (z. B. Ausgabeschicht)
  • 🎯 Wandelt Rohwerte in Wahrscheinlichkeitsverteilung

\[ f(x_i) = \frac{e^{x_i}}{\sum_{j=1}^k e^{x_j}} \]

\[ 1 = \frac{\sum_{j=1}^k e^{x_j}}{\sum_{j=1}^k e^{x_j}} \]

  • e sorgt für positive differenzierbare Werte
  • Summe normalisiert (alle Werte sollen eine Wahrscheinlichkeitsverteilung bilden)
  • Resultat ist immer ein Vektor mit positiven Einträgen die in Summe 1 ergeben
  • Vektor als WK für Klassenzugehörigkeit
  • Benötigt ein Mapping (welche WK zu welcher Klasse)

Dabei ist \(k\) die Anzahl der Klassen

2.10 Gradienten und deren Aussagekraft

  • ℹ️ Der Gradient ist die Ableitung einer Funktion nach ihren Eingabewerten
  • 🎯 Zeigt Richtung des stärksten Wachstums und Einfluss der Parameter
  • 🖼️ Dargestellt durch den Nabla Operator \(\nabla\)
  • Ein Vector aus partiellen Ableitungen
  • Zeigt ebenfalls Einfluss einzelner Parameter an
  • Kann in mehrdimensionalen Bereich als Kostenlandschaft interpretiert werden bsp. später

\[ f : \mathbb{R}^2 \to \mathbb{R} \]

\[ f(x, y) = 2x^2 + 3y \]

\[ \nabla f = \begin{pmatrix} \frac{\partial f}{\partial x} \\ \frac{\partial f}{\partial y} \end{pmatrix} = \begin{pmatrix} 4x \\ 3 \end{pmatrix} \]

  • f ist eine Funktion die ein Element aus einem 2d Raum auf einen Wert abbildet
  • jedem Punkt (x, y) wird eine Reele Zahl zugeordnet
  • x größter Einfluss auf die Steigung von f
  • y konstant positiven Einfluss (Steigung als f’)
  • je größer x wird desto horizontaler wird der Vektor (bspw. unterschied x = 1000 und x = 1001)

\[ \nabla f = \begin{pmatrix} \frac{\partial f}{\partial x} \\ \frac{\partial f}{\partial y} \end{pmatrix} = \begin{pmatrix} 4x \\ 3 \end{pmatrix} \]

  • ℹ️ \(x\) hat den größten Einfluss auf die Steigung von \(f\)
  • ℹ️ \(y\) hat konstant positiven Einfluss auf die Funktion
  • ℹ️ \(\nabla f\): Richtung des größten Wachstums
  • ℹ️ \(-\nabla f\): Richtung der größten Abnahme

2.11 Kostenfunktionen (Loss Function)

  • ℹ️ Messen den Unterschied zwischen Schätzung \(\hat{y}\) und Zielwert \(y\)
  • ℹ️ Bei Klassifikationsproblemen eignet sich Cross Entropy Loss
  • 🎯 Fehlerwert durch Anpassung der Gewichte zu minimieren
  • \(\hat{y}\): Schätzer, \(y\): Wert (Ermitteln Differenz)
  • Wird zur Anpassung von Gewichten verwendet (zsm. mit Gradienten)
  • Ziel beim Training : Soll iterativ reduziert werden

2.12 Gewichtsanpassung

  • ℹ️ Nutzt den Gradienten der Kostenfunktion, um zu bestimmen, wie stark jedes Gewicht beiträgt
  • ℹ️ Beruht auf der Kettenregel der Ableitung
  • 🎯 Fehlerwert durch Anpassung der Gewichte zu minimieren
  • Grundlage: Neuronen, FeatureMaps, etc. werden als Funktionen interpretiert
  • IN NNs sind eingaben der Funktion Gewichte, Ausgabe Vorheriger Neuronen (ebenfalls Funktionen) und Bias
  • Durch Innere Funktionen (Neuron -> Neuron (evtl. mit Aktivierungsfunktion)) kann partiell bis zu den Gewichten abgeleitet werden
  • Einfluss einzelner Paramter zur Steigung der Kostenfunktion kann ermittelt werden

Gradient Descent

  • ℹ️ Iterativer Optimierungsalgorithmus
  • 🎯 Gewichte in Richtung des negativen Gradienten Bewegen

\[ w_i \gets w_i - \eta \cdot \frac{\partial L}{\partial w_i} \]

  • eta ist Lernrate
  • Pfeil (gets) steht für Zuweisung / Aktualisierung (wie wird das neue wi gesetzt)
  • Das Gewicht wi wird in Richtung des negativen Gradienten der Kostenfunktion angepasst

\(\eta\) ist die Lernrate

Kettenregel

\[ f(x) = u(v(x)) \]

\[ f'(x) = u'(v(x)) v'(x) \]

  • wichtig: äußere mal innere Ableitung
  • verschachtelte Funktionen: Rückschluss zu Neuronen später Schichten

Reminder

Kettenregel

\[ L(\sigma(z_6)) \]

\[ \frac{dL}{dz_6} = \frac{dL}{d \sigma (z_6)} \frac{d \sigma (z_6)}{dz_6} \]

  • d: totale Ableitung (berücksichtigt alle Abhängigkeiten)
  • del (Partialzeichen): partielle Ableitung (berücksichtig eine Variable alle anderen fix)
  • Hier egal weil simples Beispiel mit einer Variable

Problem: \(z_6(\sigma (z_5))\), \(z_5(\sigma (z_4))\), …

\[ \sigma(x) = \frac{1}{1 + e^{-x}} \]

\[ \frac{d\sigma}{dx} = \sigma(x)(1-\sigma(x)) \]

  • Hier das Problem erwähnen sigmoud Ableitung in dem Produkt (aus Kettenregel)
  • Sigmoud Ableitung sehr schnell nahe null
  • Viele null werte im Produkt = wenig Gewichtskorrektur

    • Biopsie zur Sicherung der Diagnose wird nicht ersetzt
  • Nachvollziehbarkeit der Features teilweise möglich: Gradient-weighted Class Activation Mapping (Grad-CAM)
  • Video zum Verfahren: Grad-CAM

Abbildung 27: Intelligent Total Body Scanner for Early Detection of Melanoma

  • Ganzkörpersscanner entwickelt von Bosch
  • Scannt den gesamten Körper in 6 min
  • Erstellt Risikobewertung für jedes einzelne “Muttermal”
  • Nutzt KI-Modelle und XAI-Methoden des Fraunhofer Heinrich-Hertz-Institut [8]
  • Scans können oft über einen Zeitraum hinweg wiederholt werden um entwicklungen zu betrachten
  • XAI-Methoden -> auflösung der KI-Blackbox
    • Concept Relevance Propagation (CRP): Zeigt, welche gelernten Konzepte (z.B. “asymmetrische Form”) für die KI-Entscheidung wichtig waren, anstatt nur Pixel hervorzuheben.
      • die KI diese Entscheidung hauptsächlich aufgrund der Konzepte “unregelmäßige Ränder” und “ungleichmäßige Pigmentierung” getroffen hat
    • Prototypical-Concept-based eXplanations (PCX): Erklärt Entscheidungen, indem es relevante Konzepte durch konkrete Beispiele (Prototypen) aus den Trainingsdaten visualisiert.
      • PCX präsentiert dem Arzt dann konkrete Beispielbilder aus den Trainingsdatenbanken von anderen, bereits bestätigten Hautkrebsfällen
    • Reveal to Revise (R2R): Ermöglicht eine interaktive Erklärung, indem es zeigt, welche Änderungen an den Eingabedaten nötig wären, um die KI-Entscheidung zu beeinflussen oder zu revidieren.
      • Wenn dieser Bereich des Muttermals symmetrischer wäre und die Farbe hier heller und gleichmäßiger, dann würde die KI ihn als ‘harmlos’ einstufen.
